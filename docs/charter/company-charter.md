# Company Charter

更新日期：`2026-03-22`

## Divergent Hypotheses

1. 这只是一个未来写代码用的普通仓库。
2. 这是一套“AI 员工提示词集合”。
3. 这是一套 AI-first 公司操作系统，先定义治理、记忆、流程，再逐步长出产品实现。

## First Principles Deconstruction

- 早期项目最稀缺的不是开发速度，而是高质量判断。
- AI-first 团队最先失控的通常不是代码，而是上下文、权限和责任边界。
- Founder 想少介入是合理的，但不能放弃方向设定权和最终拍板权。
- 编码前最该固化的资产是：治理结构、文档分类、决策机制、记忆系统、自动化边界。

## Convergence to Excellence

我们采纳第 3 条路径：

**先做操作系统，再做产品。**

因此本仓库当前定义为：

> 一个为 `trading-agent` 产品准备的 AI-native Operating System。

## 当前阶段目标

1. 建立最小管理层 org chart。
2. 建立阶段门式 decision workflow。
3. 建立跨工具可复用的记忆与文档体系。
4. 建立 Codex / Claude Code 的项目级 agents、skills、hooks、rules 基线。
5. 让 Founder 只在方向性问题与最终拍板时介入。

## 当前阶段非目标

1. 不承诺交易胜率。
2. 不直接实现交易策略引擎。
3. 不搭全自动高权限 agent orchestra。
4. 不在没有清晰 artifact 规范前引入复杂 CI 自动化。

## 工作守则

1. 一个主题一个 owner。
2. 反对意见必须制度化存在。
3. 任何决策必须带证据、反证、风险与 tradeoff。
4. 没有回写 memory 的口头共识无效。
5. 自动化默认从低风险动作开始，权限逐层放大。
